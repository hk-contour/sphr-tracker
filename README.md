# sphr-tracker

Demand-signal monitor for SPHR's Wizard of Oz residency at the Sphere, Las Vegas.

Pulls public + scraped ticket data daily, generates a single-page HTML dashboard with the leading indicators that map to the SPHR thesis.

## What it tracks

| Source | What we get | Cost |
|---|---|---|
| **SeatGeek API** | Full schedule of upcoming shows + Ticketmaster event URLs | Free |
| **Google Trends** (via Playwright) | Weekly search-volume index for the keyword basket | Free |
| **Ticketmaster Discovery API** | Status code (`onsale`/`offsale`/`cancelled`), sales windows | Free |
| **Ticketmaster seatmap** (via ScrapingBee) | Face price range + 40 "best available" quickpicks (section, row, price) per show | Paid |

Pricing data and seat-level availability are not publicly accessible through any free API for this residency — they sit behind Ticketmaster's bot defense (Imperva) and require a stealth-proxy bypass. See `docs/findings.md` (TODO) for the full reachability map.

## Setup

```bash
# 1. Install dependencies
pip install --user requests beautifulsoup4 pandas playwright pytrends
python3 -m playwright install chromium

# 2. Create .env with the four keys
cat > .env <<EOF
SEATGEEK_CLIENT_ID=<from seatgeek.com/build>
SEATGEEK_SECRET=<from seatgeek.com/build>
TM_DISCOVERY_API_KEY=<from developer.ticketmaster.com>
SCRAPINGBEE_API_KEY=<from scrapingbee.com>
EOF

# 3. Run all pulls
bash pulls/seatgeek.sh
python3 pulls/pytrends_pull.py
python3 pulls/tm_discovery.py
python3 pulls/tm_seatmap.py --days 30

# 4. Build the dashboard
python3 dashboard.py
open dashboard.html
```

## Daily cadence

The intended run pattern (set up via `launchd` or GitHub Actions):

| Pull | Frequency | Approx cost |
|---|---|---|
| SeatGeek schedule | Weekly | $0 |
| Google Trends | Daily | $0 |
| TM Discovery | Daily | $0 (within 5k req/day free tier) |
| TM Seatmap (ScrapingBee) | Daily, ~70 events in 30-day forward window | ~$99/mo (ScrapingBee Freelance plan) |

Each daily ScrapingBee batch burns ~5,250 credits (~70 events × 75 credits/event with `stealth_proxy=true`). Monthly: ~157,500 credits → fits Freelance plan ($99/mo, 250k credits) with headroom.

## What signals come out of this

- **Capacity-fill curve** (per-show, T-30 → T-0 days): tracks how the "best available" price drifts and which sections still have low-tier inventory. A show whose best-available climbs as the date approaches is selling out; one that holds steady or drops is softening.
- **Cohort drift**: shows in a given calendar week, compared against shows in the prior week at the same T-N lead time.
- **Status flips**: TM Discovery `onsale → offsale` transitions = real sellout or sales close events.
- **Search interest YoY**: Google Trends as a top-of-funnel proxy.

The trade-able output is the *direction* and *drift* of these signals — not the absolute level, since held inventory (broker holds, ADA, sponsor blocks) biases the public data.

## Known limitations

- ScrapingBee scraping of Ticketmaster violates TM's ToS. Compliance review at Contour is recommended before enabling daily polling.
- SeatGeek's API no longer returns pricing/listing data (deprecated ~2023).
- TM Discovery does not expose face pricing for film residencies — only status codes.
- TripAdvisor scrape is not yet built; same ScrapingBee path that unlocks TM would unlock it.
